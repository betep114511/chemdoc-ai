# ChemDoc AI PRD

## 1. 背景

化学实验室中的知识通常分散在论文、实验 SOP、试剂 SDS、组会记录和师兄师姐经验中。新人在执行实验前，需要反复查找反应条件、试剂风险、异常处理方式和废液要求。通用大模型虽然能解释概念，但容易出现无来源回答，在实验安全场景中风险较高。

ChemDoc AI 的目标不是做一个万能化学聊天机器人，而是做一个“可追溯、可拒答、可反馈”的实验知识助手。

## 2. 目标用户

### 核心用户

- 化学/材料方向研究生：需要快速理解论文、复现实验、查询 SOP。
- 实验室新人：不熟悉试剂风险和实验室内部流程。
- 研发助理：需要按标准流程整理实验条件和安全检查项。

### 次级用户

- 导师/PI：希望降低新人重复问答和实验风险。
- 实验室管理员：希望统一管理 SOP、SDS 和安全要求。

## 3. 用户痛点

| 痛点 | 具体表现 | 影响 |
| --- | --- | --- |
| 文档分散 | 论文、SOP、SDS 存在不同文件夹和聊天记录中 | 查找耗时长 |
| 信息难读 | 文献和 SDS 篇幅长，关键信息不明显 | 新人理解成本高 |
| AI 不可信 | 通用模型可能编造实验条件 | 高风险场景不敢用 |
| 经验难沉淀 | 异常处理依赖口口相传 | 复现实验效率低 |
| 安全易遗漏 | PPE、废液、禁忌混放容易漏看 | 带来合规和安全风险 |

## 4. 产品目标

### 业务目标

- 将实验前查找 SOP/安全信息的平均耗时降低 40%。
- 将新人重复提问和低质量实验准备减少。
- 建立实验室内部知识库和反馈闭环。

### 产品目标

- 用户能在 3 分钟内基于上传文档获得有引用来源的回答。
- 高风险问题必须展示风险提示或触发人工确认。
- 用户反馈能沉淀为后续评测集和知识库更新任务。

### 非目标

- 不自动生成危险实验的完整操作方案。
- 不替代导师、EHS 或实验室安全负责人。
- 不承诺对无资料支撑的问题给出确定答案。

## 5. MVP 范围

### P0 功能

1. 文档上传与管理
   - 支持 PDF、Markdown、纯文本。
   - 文档类型：论文、SOP、SDS、实验记录。
   - 用户可按项目、试剂、实验类型打标签。

2. RAG 问答
   - 用户输入自然语言问题。
   - 系统检索相关文档片段。
   - 回答必须包含引用来源、页码或段落位置。
   - 无可靠来源时拒答并提示补充资料。

3. 安全提示
   - 识别强酸、强碱、氧化剂、易燃溶剂、有毒试剂等关键词。
   - 涉及高风险操作时追加 PPE、通风橱、废液分类等提示。
   - 高风险建议需要人工确认。

4. 用户反馈
   - 有用、无用、有风险、引用不对。
   - 反馈样本进入失败案例池。

5. 质量看板
   - 引用命中率、幻觉率、拒答正确率、人工接管率、平均响应延迟。

### P1 功能

- 实验前安全 checklist 自动生成。
- 文献实验条件结构化抽取。
- 多文档对比：同一反应条件在不同论文中的差异。
- 知识库过期提醒。

### P2 功能

- 与 ELN/LIMS 系统集成。
- 团队权限管理。
- 支持导师审核和 SOP 版本发布流。

## 6. 核心用户流程

### 流程 A：新人查询 SOP

1. 用户选择项目知识库。
2. 输入问题：“配置 1M 盐酸时需要注意什么？”
3. 系统检索 SOP 和 SDS。
4. 输出回答：结论、依据、风险提示、引用来源。
5. 用户点击“加入实验前检查清单”。

### 流程 B：论文复现实验

1. 用户上传论文。
2. 系统识别实验部分。
3. 用户提问：“这篇论文中聚合反应的温度和时间是什么？”
4. 系统回答并附原文引用。
5. 用户保存为实验条件卡片。

### 流程 C：AI 回答纠错

1. 用户发现引用不准确。
2. 点击“引用不对”。
3. 选择错误类型：检索失败、理解错误、上下文不足、安全风险。
4. 系统将样本加入回归测试集。
5. PM/运营定期复盘失败样本，推动知识库或 Prompt 迭代。

## 8. AI 方案

### 推荐技术路线

- 文档解析：PDF 文本抽取 + 章节识别 + 表格抽取。
- 切片策略：按标题、段落和语义块切分，保留页码和文档来源。
- 检索：Embedding 语义检索 + 关键词召回混合检索。
- 重排：根据问题、文档类型和安全关键词进行 rerank。
- 生成：模型只基于检索片段回答，必须输出引用。
- 安全层：规则引擎识别高风险关键词，触发提示、拒答或人工确认。

### 回答结构

```text
结论：直接回答用户问题。
依据：列出来自哪些文档和片段。
风险提示：涉及危险品、加热、压力、强腐蚀等场景时展示。
下一步：建议用户查看 SOP、咨询导师或补充文档。
置信度：高 / 中 / 低。
```

### 拒答策略

系统在以下情况拒答：

- 没有检索到可靠文档来源。
- 用户要求绕过安全规范。
- 用户询问危险化学品的不当制备、伤害性用途或规避监管。
- 问题需要专业人员现场判断。

## 9. 风险与兜底

| 风险 | 可能后果 | 兜底策略 |
| --- | --- | --- |
| 模型幻觉 | 输出错误实验条件 | 强制引用、无来源拒答 |
| 检索失败 | 漏掉关键安全要求 | 高风险关键词触发人工确认 |
| 文档过期 | 使用旧 SOP | 文档版本管理和过期提醒 |
| 用户误用 | 把 AI 当最终决策者 | 显示责任边界和确认流程 |
| 数据泄露 | 内部论文或配方外泄 | 权限、私有部署、日志脱敏 |

## 10. 指标

### 北极星指标

每周成功完成的“有引用实验知识查询”次数。

### 产品指标

- 查询完成率
- 用户满意度
- 平均查找耗时
- 文档上传量
- 反馈率

### AI 质量指标

- 引用命中率
- 答案准确率
- 幻觉率
- 拒答正确率
- Top-K 检索召回率
- 高风险漏报率

### 工程与成本指标

- P95 响应延迟
- 单次查询 token 成本
- 检索失败率
- 系统错误率

## 11. 上线计划

### 第 1 阶段：可用原型

- 目标：证明用户愿意用引用式 AI 查询实验文档。
- 范围：上传样例 SOP/SDS，支持 20 个常见问题。
- 验证：5 名化学背景用户试用，收集反馈。

### 第 2 阶段：小范围试点

- 目标：验证 RAG 回答质量和安全机制。
- 范围：1 个课题组或 1 个项目知识库。
- 验证：50 条黄金评测集，准确率和引用命中率达标。

### 第 3 阶段：团队版

- 目标：形成实验室知识管理闭环。
- 范围：多人权限、SOP 版本、反馈回流、质量看板。

## 12. 验收标准

- 80% 以上 P0 问题能找到引用来源。
- 高风险问题 100% 展示安全提示或人工确认。
- 无来源问题拒答正确率不低于 90%。
- 用户能在 3 分钟内完成一次查询、查看引用并提交反馈。
