# ChemDoc AI 指标与评测方案

## 1. 指标设计原则

ChemDoc AI 属于高可信知识问答产品，不能只看 DAU、留存和满意度。AI PM 需要同时关注：

- 用户是否完成任务。
- 回答是否有文档依据。
- 高风险场景是否被正确识别。
- 模型错误是否能被发现、止损和迭代。
- 成本和延迟是否适合真实使用。

## 2. 北极星指标

每周成功完成的“有引用实验知识查询”次数。

定义：

- 用户提出实验/文献/SOP/SDS 相关问题。
- 系统给出回答，并至少包含 1 个有效引用来源。
- 用户没有标记“有风险”或“引用不对”。

选择原因：

- 它同时代表使用频次、任务完成和可信回答。
- 避免只追求聊天次数，导致低质量 AI 输出变多。

## 3. 产品指标

| 指标 | 定义 | 目标 |
| --- | --- | --- |
| 查询完成率 | 用户提出问题后获得可用回答的比例 | MVP >= 70% |
| 平均查找耗时 | 从提问到获得可用答案的时间 | 比人工查找下降 40% |
| 用户满意度 | 有用反馈 / 总反馈 | MVP >= 75% |
| 反馈率 | 用户对回答进行标注的比例 | MVP >= 20% |
| 文档上传量 | 每个知识库上传文档数量 | 试点 >= 20 份 |
| 复用率 | 用户再次引用已保存答案/清单的比例 | 后续观察 |

## 4. AI 质量指标

| 指标 | 定义 | 目标 |
| --- | --- | --- |
| 引用命中率 | 回答中的引用是否支持结论 | MVP >= 85% |
| 答案准确率 | 回答与标准答案一致的比例 | MVP >= 80% |
| 幻觉率 | 回答中出现无依据信息的比例 | MVP <= 5% |
| 拒答正确率 | 无来源/高风险问题被正确拒答的比例 | MVP >= 90% |
| Top-K 检索召回率 | 标准答案来源是否出现在前 K 个检索片段中 | MVP >= 85% |
| 高风险漏报率 | 高风险问题未触发提示的比例 | MVP = 0% 作为上线目标 |

## 5. 体验与成本指标

| 指标 | 定义 | 目标 |
| --- | --- | --- |
| P95 响应延迟 | 95% 查询的响应时间 | <= 8 秒 |
| 单次 token 成本 | 每次问答消耗成本 | 按预算设门槛 |
| 二次追问率 | 用户需要追问才能完成任务的比例 | 用于发现答案不完整 |
| 人工接管率 | 需要导师/管理员确认的比例 | 高风险场景允许较高 |

## 6. 评测集设计

### 样本结构

每条样本包含：

- `case_id`：样本 ID。
- `scenario`：文献查询、SOP 查询、SDS 安全、异常处理、拒答。
- `risk_level`：low / medium / high。
- `question`：用户问题。
- `expected_behavior`：应答、拒答、提示人工确认。
- `must_cite`：必须引用的文档。
- `gold_answer`：标准答案要点。
- `failure_tags`：可能失败类型。

### 样本覆盖

| 类型 | 数量占比 | 示例 |
| --- | --- | --- |
| 文献实验条件查询 | 25% | 温度、时间、溶剂、材料比例 |
| SOP 操作流程查询 | 25% | 配液、仪器使用、废液处理 |
| SDS 安全查询 | 25% | PPE、储存、不相容物质 |
| 异常处理 | 15% | 沉淀、变色、温度异常 |
| 拒答/安全边界 | 10% | 无文档来源、危险不当用途 |

## 7. 评测方式

### 自动评测

- 规则检查：是否有引用、格式是否完整、是否触发高风险提示。
- 文本匹配：关键字段是否覆盖，如 PPE、通风橱、废液分类。
- 检索评测：标准来源是否在 Top-K 中。

### LLM-as-Judge

用于评估：

- 答案是否支持问题。
- 引用是否足以支撑结论。
- 是否加入了无依据信息。
- 风险提示是否充分。

注意：

- LLM-as-Judge 只能作为辅助。
- 高风险样本必须人工抽检。

### 人工评测

由化学背景用户或实验室管理员抽检：

- 高风险问题 100% 人审。
- 中风险问题 30% 抽检。
- 低风险问题 10% 抽检。

## 8. 上线门禁

MVP 试点上线前必须满足：

- 引用命中率 >= 85%。
- 幻觉率 <= 5%。
- 拒答正确率 >= 90%。
- 高风险漏报率 = 0%。
- P95 响应延迟 <= 8 秒。
- 至少 5 名目标用户完成试用。

如果未达标：

- 引用命中率低：优化切片、检索、rerank。
- 幻觉率高：收紧 prompt，要求只基于引用回答。
- 拒答正确率低：加入拒答分类器或规则。
- 高风险漏报：补充安全词典和规则拦截。

## 9. 失败归因

| 错误类型 | 说明 | 修复方向 |
| --- | --- | --- |
| 检索失败 | 正确文档未被召回 | 优化切片、关键词召回、embedding |
| 引用不支持 | 找到文档但引用无法支撑回答 | rerank、引用校验 |
| 模型幻觉 | 模型加入无来源内容 | prompt 约束、拒答策略 |
| 答案不完整 | 遗漏关键安全点 | 标准答案、模板、规则补充 |
| 安全漏报 | 危险场景未提示 | 安全词典、规则引擎 |
| 过度拒答 | 可回答问题被拒绝 | 调整置信度阈值 |

## 10. 迭代闭环

1. 用户提交反馈。
2. 系统记录问题、检索片段、回答、引用和用户反馈。
3. PM 每周分析失败案例。
4. 将典型失败样本加入回归测试集。
5. 对 Prompt、知识库、切片和安全规则做版本更新。
6. 新版本必须通过回归测试再灰度上线。

## 11. 面试表达要点

可以这样讲：

> 我没有把 ChemDoc AI 的指标停留在 DAU 或满意度，而是专门设计了 AI 质量指标和上线门禁。因为化学实验场景对可信度要求很高，所以我重点看引用命中率、幻觉率、拒答正确率和高风险漏报率。上线前用黄金评测集做离线评测，上线后把用户差评和高风险案例回流到测试集，形成评测驱动迭代。
